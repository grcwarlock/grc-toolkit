1. What is the GRC Toolkit?

The GRC Toolkit is an open-source platform that helps security and compliance teams manage governance, risk, and compliance from a single system. Instead of tracking controls in spreadsheets and collecting evidence manually, the toolkit automates evidence collection across cloud providers, evaluates policies continuously, and produces audit-ready reports.

  • 437 compliance controls across NIST 800-53, ISO 27001, and SOC 2
  • Automated evidence collection from AWS, Azure, and GCP using native APIs
  • 251 OPA/Rego policies that enforce compliance rules continuously
  • Monte Carlo risk analysis that quantifies risk in dollars, not guesses
  • 521 framework crosswalks so you assess once and report across multiple standards
  • 35+ tool integrations including CrowdStrike, Splunk, Okta, Tenable, and more
  • Customer trust portal for prospects to see your compliance posture
  • React dashboard with real-time charts, trend analysis, and remediation guidance

2. Quick Start

Get the toolkit running on your machine in under five minutes. This setup uses SQLite, so you don't need PostgreSQL or Docker.

Prerequisites

  • Python 3.11 or later
  • Node.js 18 or later (only if modifying the frontend)
  • Git

Step 1 — Clone & install

$ git clone https://github.com/your-org/grc-toolkit.git $ cd grc-toolkit $ python3 -m venv .venv $ source .venv/bin/activate $ pip install -r requirements.txt

Step 2 — Configure environment

$ cp .env.example .env

Open .env and set these values:

# Database (SQLite for local development) GRC_DATABASE_URL=sqlite:///grc_toolkit.db # JWT signing key (any random string for local dev) GRC_SECRET_KEY=any-random-string-for-local-dev # Leave empty to disable API key auth in dev mode GRC_API_KEYS=

Step 3 — Load demo data

$ python -m scripts.seed_demo_data

This creates demo users, sample assessments for 5 frameworks, 15 integrations, 20 evidence records, 15 vendors, and 10 policy violations.

Step 4 — Start the server

$ uvicorn api.main:app --reload --host 0.0.0.0 --port 8000

Open http://localhost:8000 in your browser.

Request Flow

Every API request passes through these layers, in order:

  1. Rate Limiter — checks if the client has exceeded their quota (120 requests/min per IP)
  2. Audit Logger — records the request with a unique ID, timestamp, method, path, and client IP
  3. Security Headers — adds HSTS, X-Frame-Options, and other protective headers
  4. CORS — validates the request origin against your whitelist
  5. Authentication — verifies the JWT token or API key
  6. Route Handler — executes business logic and returns JSON
Every Request Gets Tracked

All responses include an X-Request-Id header for tracing. Sensitive headers (Authorization, API keys, cookies) are automatically redacted in all logs.

The app uses SQLAlchemy 2.0 with 10 tables. All primary keys are UUIDs (stored as strings for SQLite/PostgreSQL portability). Tables are created automatically on startup via init_db().

Key Indexes

  • evidence_records: control_id, provider, account_id, collected_at
  • assessment_results: run_id, control_id, status
  • users: email (unique index)
  • policy_violations: policy_id, status

Key Relationships

  • assessment_results.run_id references assessment_runs.id (cascade delete)
  • Evidence records are linked to assessment results via evidence_ids JSON array
  • Audit logs reference any resource via resource_type + resource_id polymorphism

10. Production Configuration

Reverse Proxy

The API binds to localhost:8000 and should never be directly exposed to the internet. Put a reverse proxy in front of it.

Nginx Configuration

server { listen 443 ssl http2; server_name grc.example.com; ssl_certificate /etc/ssl/certs/grc.crt; ssl_certificate_key /etc/ssl/private/grc.key; location / { proxy_pass http://127.0.0.1:8000; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; } }

Security Headers

Every response includes these protective headers automatically:

HeaderValueWhat It Prevents
X-Content-Type-OptionsnosniffMIME-type confusion attacks
X-Frame-OptionsDENYClickjacking
X-XSS-Protection1; mode=blockCross-site scripting
Strict-Transport-Securitymax-age=31536000HTTP downgrade attacks
Permissions-Policygeolocation=(), camera=()Unwanted browser API access

Rate Limiting

  • Algorithm: Token bucket, per-IP tracking
  • Default: 120 requests/min, 20 burst capacity
  • Exceeded: Returns HTTP 429 with Retry-After header
  • Performance: O(1) per request, no database queries (in-memory)

Input Validation

  • All request payloads validated via Pydantic v2 schemas with type coercion
  • Enum fields restricted to known-good value sets
  • Vendor updates use a field allowlist to prevent mass assignment
  • SQLAlchemy ORM prevents SQL injection via parameterized queries

Audit Trail

  • Every request logged: timestamp, request ID, method, path, client IP, status, duration
  • Sensitive headers (Authorization, API keys, cookies) redacted automatically
  • Database changes tracked in audit_logs table with before/after JSON snapshots
  • All responses include X-Request-Id for traceability

12. Policy-as-Code (OPA)

The toolkit uses Open Policy Agent to continuously evaluate your infrastructure against compliance rules written in Rego. With 251 policy files, every control across all three major frameworks has enforcement coverage.

How It Works

  1. Client sends resource data to POST /api/v1/policies/evaluate
  2. API forwards the data to OPA at http://opa:8181
  3. OPA evaluates the matching Rego policies against the resource data
  4. Violations are returned and stored in the policy_violations table
  5. Unresolved violations appear on the dashboard and trigger Slack/email alerts

18. Extending the Toolkit

Add a New Control Assertion

  1. Define the check in config/frameworks.yaml under the appropriate control
  2. Write the assertion function in modules/control_assessor.py
  3. Create a Rego policy in policies/<framework>/<family>/
  4. Add test coverage in tests/

Add a New Connector

  1. Subclass BaseConnector in modules/connectors/
  2. Implement validate_config(), collect(), and health_check()
  3. Register with ConnectorRegistry
  4. Add credential fields to api/routers/tool_config.py

Add a New API Router

  1. Create the router file in api/routers/<name>.py
  2. Import and register in api/main.py with app.include_router()
  3. Add Pydantic schemas in api/schemas.py
  4. Add route tests in tests/test_api.py

Add a New Framework

  1. Create config/<framework_key>.yaml with control definitions
  2. Add crosswalk entries to config/crosswalks.yaml
  3. Create Rego policies in policies/<framework>/
  4. Add assertions in modules/control_assessor.py
  5. Add test coverage
